Next steps

  • Fit models on training data

  • Make predictions on testing data

  • Evaluate predictions on testing data:

    • Linear models: R-squared, adjusted R-squared, RMSE (root mean squared error), etc.
    • Logistic models: False negative and positive rates, AUC (area under the curve), etc.

  • Make decisions based on model predictive performance, validity across various testing/training splits (aka ‚Äúcross validation‚Äù), explainability

Recap

  • Split data into training and testing sets (generally 75/25)

  • Fit models on training data and reduce to a few candidate models

  • Make predictions on testing data

  • Evaluate predictions on testing data using appropriate predictive performance metrics

    • Linear models: Adjusted R-squared, AIC, etc.
    • Logistic models: False negative and positive rates, AUC (area under the curve), etc.

  • Don‚Äôt forget to also consider explainability and domain knowledge when selecting a final model

  • In a future machine learning course: Cross-validation (partitioning training data into training and validation sets and repeating this many times to evaluate model predictive performance before using the testing data), feature engineering, hyperparameter tuning, more complex models (random forests, gradient boosting machines, neural networks, etc.)

Chat GPT - Take 1 response

Here‚Äôs a short, safe solution. It keeps all columns from tv, selects only rows whose title contains the exact uppercase string ‚ÄúCSI‚Äù, stores that in csi, then prints the first 10 rows.

# Base R solution (case-sensitive match for "CSI")
csi <- tv[grepl("CSI", tv$title, fixed = TRUE), ]

# Show the first 10 rows
head(csi, 10)

If you prefer dplyr:

library(dplyr)

csi <- tv %>%
 filter(grepl("CSI", title, fixed = TRUE))

csi %>% slice_head(n = 10)

Notes: The code above searches for the exact uppercase substring "CSI" (per your instructions). If you want a case-insensitive match (e.g., to also catch "csi"), use grepl("CSI", tv$title, ignore.case = TRUE) or fixed = FALSE with ignore.case = TRUE.

Guidelines and best practices for using Chat GPT

  • Do not just copy-paste the prompt ‚Äì for appropriate academic conduct, for your own learning, and for getting to better results faster
  • Engineer the prompt until the response starts to look like code you‚Äôre learning in the course
  • If the response is not correct, ask for a correction
  • If the response doesn‚Äôt follow the guidelines, ask for a correction
  • When you ask for a correction, be ready for it to ‚Äúcorrect‚Äù (i.e., change) more than you asked for
  • Do not just copy-paste code from Chat GPT responses, run it line-by-line and edit as needed ‚Äì Chat GPT does not execute R code so there‚Äôs no guarantee any of the code it gives you works
  • Watch out for clear mistakes in the response: do not keep loading packages that are already loaded, use the base pipe |>, use the tidyverse style, do not use unnecessary print() statements, be critical of code smell, etc.
